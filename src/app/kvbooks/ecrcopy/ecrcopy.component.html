KvBook ECR Copy

The ECR Copy KvBook allow you to pull docker images from Docker Hub and push them into a private ECR repository on AWS with KaiaC. Using images that are stored in ECR is quite useful since you then control the availability of the copied images.

Design

This is how an ECR Copy KvBook works

Key/Value parameters

# Name Example value Description
1 GL_AWS_REGION us-west-2 The AWS region your resources will be deployed
3 GL_AWS_CRED_FILE_PATH ~/.aws/credentials The AWS credentials file path
4 GL_AWS_CRED_PROFILE default The AWS credentials profile
5 GL_KAIAC_MODULE ecrcopy The name of kaiac module to run
6 GL_NAMESPACE dbdump001 The application global resources namespace
7 GL_NAME demo1 The application name
8 GL_STAGE staging The application stage name
9 GL_INPUT_CONFIG_FILE /ecrcopy/001_runbook_cloud_oregon_ecrcopy_laravel_config.txt The input file of docker hub images list
10 GL_DOCKER_REPOSITORY 041296962024.dkr.ecr.us-west-2.amazonaws.com The Docker repository used to push images

Results

Go to your Elastic Container Registry Service console.
Your ECR repository should contain your images

Try

You can use this KvBook to try to use an ECR Copy with KaiaC :
GL_AWS_REGION="us-west-2"
GL_AWS_CRED_FILE_PATH="~/.aws/credentials"
GL_AWS_CRED_PROFILE="default"
GL_KAIAC_MODULE="ecrcopy"
GL_NAMESPACE="ecrcopy001"
GL_NAME="demo1"
GL_STAGE="staging"
GL_INPUT_CONFIG_FILE="/ecrcopy/001_runbook_cloud_oregon_ecrcopy_laravel_config.txt"
GL_DOCKER_REPOSITORY="041292242005.dkr.ecr.us-west-2.amazonaws.com"
GL_AWS_REGION_ECR="us-west-2"
