KvBook Operations

The Operations KvBook allow you to schedule a stop and restart of your EC2 instances. To achieve it KaiaC creates two AWS Systems Manager Maintenance Windows.

Design

This is what you can create with an Operations KvBook

Key/Value parameters

# Name Example value Description
1 GL_AWS_REGION us-west-2 The AWS region your resources will be deployed
3 GL_AWS_CRED_FILE_PATH ~/.aws/credentials The AWS credentials file path
4 GL_AWS_CRED_PROFILE default The AWS credentials profile
5 GL_KAIAC_MODULE operations The name of kaiac module to run
6 GL_NAMESPACE dbdump001 The application global resources namespace
7 GL_NAME demo1 The application name
8 GL_STAGE staging The application stage name
9 GL_TARGET_DEPLOYMENT_TAG_VALUE alfresco-demo1-prod Target Deployment tag value used to create maintenance window
10 OP_MW_SCHEDULE_START cron(0 10 ? * MON-FRI *) The operations START maintenance window schedule
11 OP_MW_SCHEDULE_STOP cron(0 18 ? * MON-FRI *) The operations STOP maintenance window schedule
12 BE_LOCAL_SOURCE_FOLDER /kaiac/alfresco-demo1-prod The folder containing the application start script.

Results

Go to your Systems Manager console.
You should see your START and STOP maintenance windows tasks.

Try

You can use this KvBook to try to create Operations with KaiaC :
GL_AWS_REGION="us-west-2"
GL_AWS_CRED_FILE_PATH="~/.aws/credentials"
GL_AWS_CRED_PROFILE="default"
GL_KAIAC_MODULE="operations"
GL_NAMESPACE="opera001"
GL_NAME="demo1"
GL_STAGE="staging"
GL_TARGET_DEPLOYMENT_TAG_VALUE="alfresco-demo1-prod"
OP_MW_SCHEDULE_START="cron(0 10 ? * MON-FRI *)"
OP_MW_SCHEDULE_STOP="cron(0 18 ? * MON-FRI *)"
BE_LOCAL_SOURCE_FOLDER="/kaiac/alfresco-demo1-prod"
