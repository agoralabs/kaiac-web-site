KvBook VPC

The VPC KvBook allow you to create a VPC with KaiaC.

Design

This is what you can create with a VPC KvBook

Key/Value parameters

# Name Example value Description
1 GL_AWS_REGION us-west-2 The AWS region your resources will be deployed
2 GL_AWS_AZS us-west-2a,us-west-2b The AWS azs your resources will be deployed
3 GL_AWS_CRED_FILE_PATH ~/.aws/credentials The AWS credentials file path
4 GL_AWS_CRED_PROFILE default The AWS credentials profile
5 GL_KAIAC_MODULE vpc The name of kaiac module to run
6 GL_NAMESPACE vpc001 The application global resources namespace
7 GL_NAME demo1 The application name
8 GL_STAGE staging The application stage name
9 GL_VPC_CREATE true The Flag to create VPC or not
10 GL_VPC_ENABLE_NAT_GATEWAY true The Flag to enable NAT GATEWAY
11 GL_VPC_SINGLE_NAT_GATEWAY true The Flag for single NAT GATEWAY
12 GL_VPC_USE_PRIVATE_SUBNETS true The Flag to use private subnets if true else public subnets
13 GL_VPC_CIDR 10.0.0.0/16 The VPC cidr block
14 GL_VPC_CIDR_SUBNET1 10.0.101.0/24 The subnet1 cidr block
15 GL_VPC_CIDR_SUBNET2 10.0.102.0/24 The subnet2 cidr block
16 GL_VPC_CIDR_SUBNET_PRV1 10.0.1.0/24 The private subnet1 cidr block
17 GL_VPC_CIDR_SUBNET_PRV2 10.0.2.0/24 The private subnet2 cidr block

Results

Go to your AWS VPC Service console.
Your VPC should be created with the name GL_NAMESPACE-GL_NAME-GL_STAGE-vpc

Try

You can use this KvBook to try to create a vpc with KaiaC :
GL_AWS_REGION="us-west-2"
GL_AWS_CRED_FILE_PATH="~/.aws/credentials"
GL_AWS_CRED_PROFILE="default"
GL_KAIAC_MODULE="vpc"
GL_NAMESPACE="vpc001"
GL_NAME="demo1"
GL_STAGE="staging"
GL_AWS_AZS="us-west-2a,us-west-2b"
GL_VPC_CREATE="true"
GL_VPC_ENABLE_NAT_GATEWAY="true"
GL_VPC_SINGLE_NAT_GATEWAY="true"
GL_VPC_USE_PRIVATE_SUBNETS="true"
GL_VPC_CIDR="10.0.0.0/16"
GL_VPC_CIDR_SUBNET1="10.0.101.0/24"
GL_VPC_CIDR_SUBNET2="10.0.102.0/24"
GL_VPC_CIDR_SUBNET_PRV1="10.0.1.0/24"
GL_VPC_CIDR_SUBNET_PRV2="10.0.2.0/24"
