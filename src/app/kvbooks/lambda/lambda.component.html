KvBook Lambda

The Lambda KvBook allow you to create a Lambda function, and expose it through an API Gateway endpoint.

Design

This is what you can create with a Lambda KvBook

Key/Value parameters

# Name Example value Description
1 GL_AWS_REGION us-west-2 The AWS region your resources will be deployed
3 GL_AWS_CRED_FILE_PATH ~/.aws/credentials The AWS credentials file path
4 GL_AWS_CRED_PROFILE default The AWS credentials profile
5 GL_KAIAC_MODULE lambda The name of kaiac module to run
6 GL_NAMESPACE dbdump001 The application global resources namespace
7 GL_NAME demo1 The application name
8 GL_STAGE staging The application stage name
9 GL_AWS_ROUTE53_DOMAIN skyscaledev.com The Route53 Domain name
10 GL_DNS_RECORD_TTL 300 The DNS record TTL
11 LA_NAME send_command The lambda function name
12 LA_RUNTIME nodejs16.x The lambda function runtime
13 LA_TIMEOUT 30 The lambda function timeout
14 LA_STAGE staging The lambda function stage name
15 LA_APIGW_SUBDOMAIN lamb001 The lambda function API Gateway sub domain
16 LA_SOURCE_DIR /root/demo-kaiac-lambda The lambda function source directory

Results

Go to your Lambda Service console.
You should see your Lambda Function, you can try to call your function using a tool like Postman or curl in command line.

Try

You can use this KvBook to try to use Lambda with KaiaC :
GL_AWS_REGION="us-west-2"
GL_AWS_CRED_FILE_PATH="~/.aws/credentials"
GL_AWS_CRED_PROFILE="default"
GL_KAIAC_MODULE="lambda"
GL_NAMESPACE="lamb001"
GL_NAME="demo1"
GL_STAGE="staging"
GL_AWS_ROUTE53_DOMAIN="skyscaledev.com"
GL_DNS_RECORD_TTL="300"
LA_NAME="send_command"
LA_RUNTIME="nodejs16.x"
LA_TIMEOUT="30"
LA_STAGE="staging"
LA_APIGW_SUBDOMAIN="lamb001"
LA_SOURCE_DIR="/root/demo-kaiac-lambda"
