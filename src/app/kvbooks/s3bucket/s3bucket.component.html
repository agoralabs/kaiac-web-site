KvBook S3 Bucket

The S3 Bucket KvBook allow you to create an S3 Bucket to host a website without a Cloudfront Distribution with KaiaC.

Design

This is what you can create with an S3 Bucket KvBook

Key/Value parameters

# Name Example value Description
1 GL_AWS_REGION us-west-2 The AWS region your resources will be deployed
3 GL_AWS_CRED_FILE_PATH ~/.aws/credentials The AWS credentials file path
4 GL_AWS_CRED_PROFILE default The AWS credentials profile
5 GL_KAIAC_MODULE s3bucket The name of kaiac module to run
6 GL_NAMESPACE s3bucket001 The application global resources namespace
7 GL_NAME demo1 The application name
8 GL_STAGE staging The application stage name
9 GL_AWS_ROUTE53_DOMAIN skyscaledev.com The Route53 Domain name
10 GL_DNS_RECORD_TTL 300 The DNS record TTL
11 FE_CREATE_DNS true Flag to generate a Route53 DNS record for S3 website
12 FE_DNS repo-kaiac-prod The S3 website domain name prefix
13 FE_S3_BUCKET_NAME repo-kaiac-prod.skyscaledev.com The S3 bucket name for S3 website

Results

Go to your S3 Service console.
Your S3 Bucket should be created with the name specified for the website

Try

You can use this KvBook to try to create an S3 Website with KaiaC :
GL_AWS_REGION="us-west-2"
GL_AWS_CRED_FILE_PATH="~/.aws/credentials"
GL_AWS_CRED_PROFILE="default"
GL_KAIAC_MODULE="s3website"
GL_NAMESPACE="s3website001"
GL_NAME="demo1"
GL_STAGE="staging"
GL_AWS_ROUTE53_DOMAIN="skyscaledev.com"
GL_DNS_RECORD_TTL="300"
FE_CREATE_DNS="true"
FE_DNS="demo1-s3website"
FE_S3_BUCKET_NAME="demo1-s3website.skyscaledev.com"
