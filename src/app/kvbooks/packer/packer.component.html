KvBook Packer

The Packer KvBook allow you to create an Amazon Machine Image with KaiaC.

Design

This is what you can create with a Packer KvBook

Key/Value parameters

# Name Example value Description
1 GL_AWS_REGION us-west-2 The AWS region your resources will be deployed
2 GL_AWS_CRED_FILE_PATH ~/.aws/credentials The AWS credentials file path
3 GL_AWS_CRED_PROFILE default The AWS credentials profile
4 GL_KAIAC_MODULE packer The name of kaiac module to run
5 GL_NAMESPACE ami001 The application global resources namespace
6 GL_NAME demo1 The application name
7 GL_STAGE staging The application stage name
8 BE_PATH_EC2_PLAYBOOK /usr/kaiac/playbooks/backend.yml The Ansible playbook used to initialize the EC2 instance
9 GL_AMI_FILTER_OWNER 099720109477 The Owner to specify for AMI filter
10 GL_AMI_FILTER_NAME ubuntu/images/hvm-ssd/ubuntu-focal-20.04-amd64-server-* The Name to specify for AMI filter
11 BE_AMI_NAME ubuntu-docker-packer The ec2 instance Amazon AMI name to be created

Results

Go to your AWS EC2 Service console.
Your AMI should be created with the name you specified

Try

You can use this KvBook to try to create an AMI with KaiaC :
GL_AWS_REGION="us-west-2"
GL_AWS_CRED_FILE_PATH="~/.aws/credentials"
GL_AWS_CRED_PROFILE="default"
GL_KAIAC_MODULE="packer"
GL_NAMESPACE="ami001"
GL_NAME="demo1"
GL_STAGE="staging"
BE_PATH_EC2_PLAYBOOK="/root/agoralabs-devops-manage/playbooks/backend.yml"
GL_AMI_FILTER_OWNER="099720109477"
GL_AMI_FILTER_NAME="ubuntu/images/hvm-ssd/ubuntu-focal-20.04-amd64-server-*"
BE_AMI_NAME="ubuntu-laravel-packer"
