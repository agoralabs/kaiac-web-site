KvBook DB Loader

The DB Loader KvBook allow you to execute an SQL Script on an existing RDS Database. It can then be usefull to load data in your database.

Design

This is how a DB Loader KvBook works

Key/Value parameters

# Name Example value Description
1 GL_AWS_REGION us-west-2 The AWS region your resources will be deployed
3 GL_AWS_CRED_FILE_PATH ~/.aws/credentials The AWS credentials file path
4 GL_AWS_CRED_PROFILE default The AWS credentials profile
5 GL_KAIAC_MODULE dbloader The name of kaiac module to run
6 GL_NAMESPACE dbload001 The application global resources namespace
7 GL_NAME demo1 The application name
8 GL_STAGE staging The application stage name
9 DB_HOST dbrds001-demo1-staging.skyscaledev.com The application database DNS alias
10 DB_NAME dbonly001 The application database name
11 DB_ADMIN_USERNAME admindbonly001 The application database Admin username
12 DB_ADMIN_PASSWORD ******** The application database Admin user password
13 DB_PORT 3306 The application database port
14 DB_SCHEMA_DUMP_PATH /dump/dbrds001.sql The db schema dump file to load PATH

Results

Use a tool like MySQL Workbench to connect to your database schema.
Then verify the data added in your database

Try

You can use this KvBook to try to use a DB Loader with KaiaC :
GL_AWS_REGION="us-west-2"
GL_AWS_CRED_FILE_PATH="~/.aws/credentials"
GL_AWS_CRED_PROFILE="default"
GL_KAIAC_MODULE="dbloader"
GL_NAMESPACE="dbload001"
GL_NAME="demo1"
GL_STAGE="staging"
DB_HOST="dbrds001-demo1-staging.skyscaledev.com"
DB_ADMIN_USERNAME="admindbonly001"
DB_ADMIN_PASSWORD="ComplexPwd"
DB_NAME="dbonly001"
DB_PORT="3306"
DB_SCHEMA_DUMP_PATH="/root/profiles/dbdump001/dump/dbrds001.sql"
