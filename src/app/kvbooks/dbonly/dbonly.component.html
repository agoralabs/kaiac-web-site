KvBook DB Only

The DB Only KvBook allow you to create a database schema inside an existing RDS instance with KaiaC.

Design

This is what you can create with a DB Only KvBook

Key/Value parameters

# Name Example value Description
1 GL_AWS_REGION us-west-2 The AWS region your resources will be deployed
2 GL_AWS_CRED_FILE_PATH ~/.aws/credentials The AWS credentials file path
3 GL_AWS_CRED_PROFILE default The AWS credentials profile
4 GL_KAIAC_MODULE dbonly The name of kaiac module to run
5 GL_NAMESPACE dbrds002 The application global resources namespace
6 GL_NAME demo1 The application name
7 GL_STAGE staging The application stage name
8 DB_HOST dbrds001-demo1-staging.skyscaledev.com The application database DNS alias
9 DB_NAME dbonly001 The application database name
10 DB_USERNAME admindbonly001 The application database Admin username
11 DB_PASSWORD ******** The application database Admin user password
12 DB_PORT 3306 The application database port

Results

Use a tool like MySQL Workbench to verify the created database schema.
Your database should be created with the name specified

Try

You can use this KvBook to try to create a DB Only with KaiaC :
GL_AWS_REGION="us-west-2"
GL_AWS_CRED_FILE_PATH="~/.aws/credentials"
GL_AWS_CRED_PROFILE="default"
GL_KAIAC_MODULE="dbonly"
GL_NAMESPACE="dbonly001"
GL_NAME="demo1"
GL_STAGE="staging"
DB_HOST="dbrds001-demo1-staging.skyscaledev.com"
DB_ADMIN_USERNAME="admindbrds001"
DB_ADMIN_PASSWORD="to be defined"
DB_NAME="dbonly001"
DB_USERNAME="admindbonly001"
DB_PASSWORD="to be defined"
DB_PORT="3306"
