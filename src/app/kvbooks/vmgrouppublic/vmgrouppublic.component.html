KvBook VM Group Public

The VM Group Public KvBook allow you to create in a public subnet, a target group behind an exisiting ALB with EC2 instances with KaiaC.

Design

This is what you can create with a VM Group Public KvBook

Key/Value parameters

# Name Example value Description
1 GL_AWS_REGION us-west-2 The AWS region your resources will be deployed
2 GL_AWS_CRED_FILE_PATH ~/.aws/credentials The AWS credentials file path
3 GL_AWS_CRED_PROFILE default The AWS credentials profile
4 GL_KAIAC_MODULE vmgrouppublic The name of kaiac module to run
5 GL_NAMESPACE vmgrpub001 The application global resources namespace
6 GL_NAME demo1 The application name
7 GL_STAGE staging The application stage name
8 GL_AWS_ROUTE53_DOMAIN skyscaledev.com The Route53 Domain name
9 GL_DNS_RECORD_TTL 300 The DNS record TTL
10 GL_VPC_FILTER_NAME tag:Name The VPC filter name
11 GL_VPC_FILTER_VALUE vpc001-demo1-staging-vpc The VPC Name
12 GL_VPC_SUBNET_FILTER_NAME tag:Name The Public Subnet filter name subnet-id or tag:Name
13 GL_VPC_SUBNET_FILTER_VALUE1 vpc001-demo1-staging-public-subnet1 The Public Subnet1 filter value eg subnet name
14 GL_VPC_SUBNET_FILTER_VALUE2 vpc001-demo1-staging-public-subnet2 The Public Subnet2 filter value eg subnet name
15 BE_PATH_EC2_PUBKEY ~/.ssh/id_rsa.pub The ec2 instance public key path
16 BE_EC2_TYPE t2.micro The EC2 instance type
17 BE_EC2_INSTANCE_COUNT 2 The EC2 instance instance count
18 BE_EC2_VOLUME_SIZE 20 The EC2 instance root volume size en giga
19 BE_LOCAL_PORT 8080 The EC2 instance local port
20 BE_AMI_NAME ubuntu/images/hvm-ssd/ubuntu-jammy-22.04-amd64-server-20230919 The ec2 instance Amazon AMI name
21 BE_CREATE_DNS true The flag to create or not a Route 53 DNS record
22 BE_DNS vmgrpub001-demo1-staging The ec2 instance domain name
23 BE_DOMAIN_NAME vmgrpub001-demo1-staging.skyscaledev.com The private subnet1 cidr block
24 BE_EXISTING_ALB_NAME alb001-demo1-staging-lb Existing ALB name to use
25 BE_EXISTING_LISTENER_443 false Existing listener to use or create if false
26 BE_ADD_SNI_CERT true Flag to add or not an sni certificate to ALB
27 GL_HEALTH_CHECK_PATH / The Application health check path in target group

Results

Go to your AWS EC2 Service console.
Your target group and EC2 instances should be created

Try

You can use this KvBook to try to create a VM Group Public with KaiaC :

GL_AWS_REGION="us-west-2"
GL_AWS_CRED_FILE_PATH="~/.aws/credentials"
GL_AWS_CRED_PROFILE="default"
GL_KAIAC_MODULE="vmgrouppublic"
GL_NAMESPACE="vmgrpub001"
GL_NAME="demo1"
GL_STAGE="staging"
GL_AWS_ROUTE53_DOMAIN="skyscaledev.com"
GL_DNS_RECORD_TTL="300"
GL_VPC_FILTER_NAME="tag:Name"
GL_VPC_FILTER_VALUE="vpc002-demo2-staging-vpc"
GL_VPC_SUBNET_FILTER_NAME="tag:Name"
GL_VPC_SUBNET_FILTER_VALUE1="vpc002-demo2-staging-public-subnet1"
GL_VPC_SUBNET_FILTER_VALUE2="vpc002-demo2-staging-public-subnet2"
BE_PATH_EC2_PUBKEY="/root/profiles/demo/ssh/SSHPUBKEY"
BE_EC2_TYPE="t2.micro"
BE_EC2_INSTANCE_COUNT="2"
BE_EC2_VOLUME_SIZE="20"
BE_LOCAL_PORT="8080"
BE_AMI_NAME="ubuntu-laravel-ami"
BE_CREATE_DNS="true"
BE_DNS="vmgrpub001-demo1-staging"
BE_DOMAIN_NAME="vmgrpub001-demo1-staging.skyscaledev.com"
BE_EXISTING_ALB_NAME="alb002-demo2-staging-lb"
BE_EXISTING_LISTENER_443="false"
BE_ADD_SNI_CERT="true"
GL_HEALTH_CHECK_PATH="/"
