KvBook Vpc Endpoint

The Vpc Endpoint KvBook allow you to create a VPC Endpoint with KaiaC. This VPC Endpoint is useful to access your EC2 instances located in a private subnet.

Design

This is what you can create with a Vpc Endpoint KvBook

Key/Value parameters

# Name Example value Description
1 GL_AWS_REGION us-west-2 The AWS region your resources will be deployed
3 GL_AWS_CRED_FILE_PATH ~/.aws/credentials The AWS credentials file path
4 GL_AWS_CRED_PROFILE default The AWS credentials profile
5 GL_KAIAC_MODULE vpcendpoint The name of kaiac module to run
6 GL_NAMESPACE tpl001 The application global resources namespace
7 GL_NAME demo1 The application name
8 GL_STAGE staging The application stage name
9 GL_VPC_FILTER_NAME tag:Name The VPC filter name
10 GL_VPC_FILTER_VALUE vpc001-demo1-staging-vpc The VPC Name
11 GL_VPC_SUBNET_FILTER_NAME tag:Name The Public Subnet filter name subnet-id or tag:Name
12 GL_VPC_SUBNET_PRV_FILTER_VALUE1 vpc001-demo1-staging-private-subnet1 The Private Subnet1 filter value eg subnet name

Results

Go to your AWS VPC Service console.
Your VPC Endpoint should be created

Try

You can use this KvBook to create a VPC Endpoint with KaiaC :
GL_AWS_REGION="us-west-2"
GL_AWS_CRED_FILE_PATH="~/.aws/credentials"
GL_AWS_CRED_PROFILE="default"
GL_KAIAC_MODULE="vpcendpoint"
GL_NAMESPACE="vpcendpt001"
GL_NAME="demo1"
GL_STAGE="staging"
GL_VPC_FILTER_NAME="tag:Name"
GL_VPC_FILTER_VALUE="vpc001-demo1-staging-vpc"
GL_VPC_SUBNET_FILTER_NAME="tag:Name"
GL_VPC_SUBNET_PRV_FILTER_VALUE1="vpc001-demo1-staging-private-subnet1"
